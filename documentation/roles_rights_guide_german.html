Eine der großen neuen Entwicklungen in Chamilo 1.7 ist das Benutzer Rollen und Rechte System. Diese Anleitung erläutert, wie man es am effektivsten einsetzen kann.

Das Benutzer Rollen und Rechte System ist neu in dieser Version. Es handelt sich um eine erste grundlegende Ausführung eines solchen Systems, und wir planen, das System über die nächsten Monate weiter zu entwickeln und zu verbessern. Wir hoffen, von Ihnen zu hören, z.B. Feedback, Vorschläge und Berichte über eventuell vorhandene Bugs. Sie können uns erreichen über das Chamilo Forum.

1. Was ist es?

Das Rollen und Rechte System gibt Chamilo viel mehr Flexibilität im Hinblick auf wer darf was an welchem Ort tun. Die grundlegenden Konzepte sind Rollen, Rechte und Orte.

1.1 Rollen

Es gibt fünf Rollen innerhalb von Chamilo. Als Kursleiter können Sie diese Rollen Benutzern in Ihrem Kurs zuweisen. Rollen sind wie Hüte, die ein Benutzer trägt, sie zeigen an, dass Benutzer einem bestimmten Typ oder einer bestimmten Gruppe angehören. Später werden wir noch die Möglichkeit hinzufügen, neue Rollen zu definieren.

  • Anonymer Gast: dies kann jeder irgendwo in der Welt sein, der mit seinem Browser Ihren Kurs besucht. Standardmäßig haben diese Benutzer keinerlei Rechte und könen nicht einmal Ihren Kurs ansehen
  • Angemeldeter Gast: das ist jemand, der auf der Chamilo Plattform angemeldet ist und Ihren Kurs besucht, ohne den Kurs belegt zu haben
  • Kursteilnehmer: das ist die Standard Rolle für Benutzer, die Ihren Kurs belegt haben. Typischerweise hat die Mehrzahl der Benutzer diese Rolle, und Kursleiter können die Rolle Lehrer Assistent einigen Benutzern zuweisen, die dann bei der Entwicklung des Kurses behilflich sein können.
  • Lehrer Assistent: die Assistent Rolle beinhaltet viele Rechte. In den meisten Werkzeugen darf der Assistent Inhalte ansehen, hinzufügen und editeren, aber nicht löschen. Der Lehrer Assistent kann standardmäßig auch die Kurseinstellungen und Kursrechte nicht ändern.
  • Kursleiter: der Kursleiter darf in seinem Kurs alles. Wer einen Kurs erstellt, erhält automatisch diese Rolle zugewiesen.

1.2 Rechte

Es gibt vier grundlegende Rechte in Chamilo:

  • Ansehen: das Recht, ein Werkzeug und seinen Inhalt anzusehen
  • Hinzufügen: das Recht, neue Inhalte in einem Werkzeug hinzuzufügen, z.B. Links, Kategorien, Dokumente, Verzeichnisse, Veröffentlichungen von Kursteilnehmern, Forum Nachrichten ...
  • Editieren: das Recht, vorhandene Inhalte zu ändern, z.B. die URL eines Links ändern, den Namen eines Verzeichnisses, Titel und Schlsselwörter eines Dokuments ändern...
  • Löschen: das Recht, vorhandene Inhalte in einem Werkzeug zu löschen, z.B. Links löschen, Veröffentlichungen von Kursteilnehmern löschen, Forum Nachrichten, Dokumente ...

1.3 Orte

Jeder Kurs und jedes Werkzeug in einem Kurs sind unterschiedliche Orte in Chamilo. Einige Beispiele für Orte sind

  • Der Kurs TEST_COURSE
  • Das Dokument Werkzeug in TEST_COURSE
  • Das Agenda Werkzeug im Kurs Psychologie 101

1.4 Rechte vergeben

Als Kursleiter bestimmen Sie, wer was an welchem Ort tun darf. Für jede Kombination von Rolle und Ort können die vier Rechte vergeben werden. Sie können z.B. das Ansehen Recht für den Lehrer Assisten im Dokumente Werkzeug vergeben, das Hinzufügen Recht für Kursteilnehmer im Links Werkzeug, usw

2. Für wen ist das von Nutzen?

Plattform Administratoren konnten in älteren Versionen nicht viel mehr tun, als Benutzern entweder Kursteilnehmer oder Kursleiter Status zu geben. Nun gibt es einige weitere Rollen, und der Plattform Admin kann genauer definieren, was Benutzer in einem Kurs ansehen, hinzufügen, editieren oder lö dürfen. Sie können standardmäßige Rechte für neu erstellte Kurse festlegen, sie können bestimmen, welche Rechte ein Lehrer Assistent, welche normale Kursteilnehmer, oder anonyme Gäste haben ...

Einige Organisationen nutzen eine externe Datenbank für Benutzer und Kurse in Chamilo. Diese können ebenfalls von diesem neuen System profitieren, z.B. indem sie Studenten, die einen Kurs regelmäßig besuchen und absolvieren, andere Rechte einräumen als denen, die einen Kurs lediglich aus Interesse belegt haben. Studenten, die an dem Kurs tatsächlich aktiv teilnehmen, könnten die Rolle "Kursteilnehmer" bekommen, während die Studenten, die lediglich aus Interesse in den Kurs kommen, als "Angemeldeter Gast" geführt werden könnten.

Kursleiter können Benutzern, die ihren Kurs belegt haben, verschiedene Rollen zuweisen. Sie können Teilnehmer zu Lehrer Assistenten machen, um ihnen mit dem Kurs zu helfen. Sie können genau festlegen, was der Assistent tun darf und was nicht. Sie können den normalen Kursteilnehmern mehr oder weniger Rechte zuweisen, für jedes Kurs Werkzeug separat. Wenn Kursteilnehmer Links und Dokumente im Kurs hinzufügen dürfen, oder neue Foren einrichten sollen, dann können Sie ihnen das Recht geben. Wenn Sie das Recht, Veröffentlichungen von Teilnehmern hochzuladen, nehmen wollen, dann können Sie auch das tun.

Studenten / Kursteilnehmer können je nach Bedarf Rechte erhalten. Sie können mehr Rechte erhalten, um einige Teile eines Kurses zu benutzen, z.B. Links gemäß ihren Interessen hinzufügen, Foren moderieren ... Sie können ebenfalls (mit oder ohne Aufsicht) auch Kurse betreuen, usw ...

3. Wie man es einfach hält

Vielleicht wollen Sie Chamilo so einfach wie möglich halten. Sie wollen nichts mit den komplexen neuen Entwicklungen wie dem Benutzer Rollen und Rechte System zu tun haben. Vielleicht haben Sie auch einfach momentan keinen Bedarf für diese Möglichkeiten.

Wir haben uns sehr bemüht, die Bediener Oberfläche von Chamilo so einfach wie möglich zu halten. Es ist weiterhin möglich, dass Lehrer und Kursleiter ihre Kurse genau so zu betreuen, wie es in älteren Chamilo Versionen der Fall war, ohne sich um das Rollen und Rechte System zu kümmern. Es verrichtet seine Aufgaben hinter der Szene. Wenn Sie dann irgendwann mehr Möglichkeiten brauchen als das alte System zur Verfügung stellt, können sie das neue System leicht erreichen.

Viele verschiedene Anwendungen sind möglich. In einigen Organisationen werden Kursleiter nur wenig damit arbeiten, Plattform Administratoren dagegen viel. In anderen werden Lehrer ausführlich mit den neuen Möglichkeiten experimentieren.

4. Wie man es benutzt

Die gute Nachricht ist: Wenn Sie einen Kurs betreuen, benutzen Sie bereits das neue System, ohne es zu wissen, sobald Sie auf ein Link klicken oder eine Seite des Kurses aufrufen. Wie können Sie nun das System am besten benutzen?

Das grundlegende Konzept ist, dass Sie für jedes Kurs Werkzeug festlegen können, wer die Inhalte dieses Werkzeugs ansehen darf, neue Inhalte und Verzeichnisse hinzufügen darf, existierende Inhalte ändern, oder Inhalte löschen darf. Sie müssen dies jedoch nicht für jeden einzelnen Benutzer angeben. Benutzer haben Rollen in Ihrem Kurs, wie etwa Angemeldeter Gast, Kursteilnehmer, Lehrer Assistent, und Kursleiter. Sie können nun jeder dieser Rollen bestimmte Rechte für jedes Werkzeug zuweisen. Sie können gestatten, dass alle Benutzer das Forum ansehen dürfen, Gäste dürfen aber da nichts hinzufügen, das bleibt Kursteilnehmern vorbehalten. Sie können festlegen, dass Ihr Assistent Dokumente hinzufügen, aber keine Dokumente löschen darf, usw.

Im Benutzer Werkzeug in Ihrem Kurs können Sie die Rollen den Benutzern zuweisen und diese editieren (einfach den Edit Link in der Liste der Benutzer klicken).

Auf der Rollen & Rechte Überblick Seite in Ihrem Kurs können Sie die Rollen von Benutzern editieren (einfach den Edit Link in der Liste der Benutzer klicken).
